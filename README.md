<p><strong>Kotlin</strong>&nbsp;(<span class="rt-commentedText nowrap"><span class="IPA nopopups noexcerpt"><a title="Help:IPA/English" href="https://en.wikipedia.org/wiki/Help:IPA/English">/<span title="/ˈ/: primary stress follows">ˈ</span><span title="'k' in 'kind'">k</span><span title="/ɒ/: 'o' in 'body'">ɒ</span><span title="'t' in 'tie'">t</span><span title="'l' in 'lie'">l</span><span title="/ɪ/: 'i' in 'kit'">ɪ</span><span title="'n' in 'nigh'">n</span>/</a></span></span>)&nbsp;is a&nbsp;<a title="" href="https://en.wikipedia.org/wiki/Cross-platform_software">cross-platform</a>,&nbsp;<a class="mw-redirect" title="Static typing" href="https://en.wikipedia.org/wiki/Static_typing">statically typed</a>,&nbsp;<a title="General-purpose programming language" href="https://en.wikipedia.org/wiki/General-purpose_programming_language">general-purpose</a>&nbsp;<a title="Programming language" href="https://en.wikipedia.org/wiki/Programming_language">programming language</a>&nbsp;with&nbsp;<a title="Type inference" href="https://en.wikipedia.org/wiki/Type_inference">type inference</a>. Kotlin is designed to interoperate fully with&nbsp;<a title="Java (programming language)" href="https://en.wikipedia.org/wiki/Java_(programming_language)">Java</a>, and the&nbsp;<a title="Java virtual machine" href="https://en.wikipedia.org/wiki/Java_virtual_machine">JVM</a>&nbsp;version of its&nbsp;<a title="Standard library" href="https://en.wikipedia.org/wiki/Standard_library">standard library</a>&nbsp;depends on the&nbsp;<a title="Java Class Library" href="https://en.wikipedia.org/wiki/Java_Class_Library">Java Class Library</a>,&nbsp;but type inference allows its&nbsp;<a title="Syntax (programming languages)" href="https://en.wikipedia.org/wiki/Syntax_(programming_languages)">syntax</a>&nbsp;to be more concise. Kotlin mainly targets the JVM, but also compiles to&nbsp;<a title="JavaScript" href="https://en.wikipedia.org/wiki/JavaScript">JavaScript</a>&nbsp;or&nbsp;<a title="Machine code" href="https://en.wikipedia.org/wiki/Machine_code">native code</a>&nbsp;(via&nbsp;<a title="LLVM" href="https://en.wikipedia.org/wiki/LLVM">LLVM</a>). Language development costs are borne by&nbsp;<a title="JetBrains" href="https://en.wikipedia.org/wiki/JetBrains">JetBrains</a>, while the Kotlin Foundation protects the Kotlin trademark.</p>
<p>On 7 May 2019, Google announced that the Kotlin programming language is now its preferred language for Android app developers.&nbsp;Since the release of&nbsp;<a title="Android Studio" href="https://en.wikipedia.org/wiki/Android_Studio">Android Studio</a>&nbsp;3.0 in October 2017, Kotlin has been included as an alternative to the standard Java compiler. The Android Kotlin compiler targets Java 6 by default, but lets the programmer choose between Java 8 to 13, for optimization.</p>

<h2><span id="History" class="mw-headline">History</span></h2>
<p>In July 2011, JetBrains unveiled Project Kotlin, a new language for the JVM, which had been under development for a year.&nbsp;JetBrains lead Dmitry Jemerov said that most languages did not have the features they were looking for, with the exception of&nbsp;<a title="Scala (programming language)" href="https://en.wikipedia.org/wiki/Scala_(programming_language)">Scala</a>. However, he cited the slow&nbsp;<a title="Compiler" href="https://en.wikipedia.org/wiki/Compiler">compilation</a>&nbsp;time of Scala as a deficiency.&nbsp;One of the stated goals of Kotlin is to compile as quickly as Java. In February 2012, JetBrains open sourced the project under the&nbsp;<a class="mw-redirect" title="Apache license" href="https://en.wikipedia.org/wiki/Apache_license">Apache 2 license</a>.</p>
<p>The name comes from&nbsp;<a title="Kotlin Island" href="https://en.wikipedia.org/wiki/Kotlin_Island">Kotlin Island</a>, near&nbsp;<a title="Saint Petersburg" href="https://en.wikipedia.org/wiki/Saint_Petersburg">St. Petersburg</a>.&nbsp;<a class="new" title="Andrey Breslav (page does not exist)" href="https://en.wikipedia.org/w/index.php?title=Andrey_Breslav&amp;action=edit&amp;redlink=1">Andrey Breslav</a>&nbsp;mentioned that the team decided to name it after an island just like Java was named after the Indonesian island of&nbsp;<a title="Java" href="https://en.wikipedia.org/wiki/Java">Java</a>&nbsp;(though the programming language Java was perhaps named after the coffee).</p>
<p>JetBrains hopes that the new language will drive&nbsp;<a title="IntelliJ IDEA" href="https://en.wikipedia.org/wiki/IntelliJ_IDEA">IntelliJ IDEA</a>&nbsp;sales.</p>
<p>Kotlin v1.0 was released on 15 February 2016.&nbsp;This is considered to be the first officially stable release and JetBrains has committed to long-term backwards compatibility starting with this version.</p>
<p>At&nbsp;<a title="Google I/O" href="https://en.wikipedia.org/wiki/Google_I/O">Google I/O</a>&nbsp;2017, Google announced first-class support for Kotlin on&nbsp;<a title="Android (operating system)" href="https://en.wikipedia.org/wiki/Android_(operating_system)">Android</a>.</p>
<p>Kotlin v1.2 was released on 28 November 2017.&nbsp;Sharing code between JVM and JavaScript platforms feature was newly added to this release (as of version 1.3 multiplatform programming is experimental).&nbsp;<a class="mw-redirect" title="Full-stack" href="https://en.wikipedia.org/wiki/Full-stack">Full-stack</a>&nbsp;demo has been made with the new Kotlin/JS Gradle Plugin.</p>
<p>Kotlin v1.3 was released on 29 October 2018, bringing coroutines for asynchronous programming.</p>
<p>On 7 May 2019, Google announced that the Kotlin programming language is now its preferred language for Android app developers.</p>
<h2><span id="Design" class="mw-headline">Design</span></h2>
<p>Development lead Andrey Breslav has said that Kotlin is designed to be an industrial-strength&nbsp;<a class="mw-redirect" title="Object oriented programming" href="https://en.wikipedia.org/wiki/Object_oriented_programming">object-oriented</a>&nbsp;language, and a "better language" than Java, but still be fully interoperable with Java code, allowing companies to make a gradual migration from Java to Kotlin.</p>
<p><a title="Semicolon" href="https://en.wikipedia.org/wiki/Semicolon">Semicolons</a>&nbsp;are optional as a&nbsp;<a title="Statement (computer science)" href="https://en.wikipedia.org/wiki/Statement_(computer_science)">statement</a>&nbsp;<a class="mw-redirect" title="Statement terminator" href="https://en.wikipedia.org/wiki/Statement_terminator#Statements">terminator</a>; in most cases a&nbsp;<a title="Newline" href="https://en.wikipedia.org/wiki/Newline">newline</a>&nbsp;is sufficient for the&nbsp;<a title="Compiler" href="https://en.wikipedia.org/wiki/Compiler">compiler</a>&nbsp;to deduce that the statement has ended.</p>
<p>Kotlin&nbsp;<a title="Variable (computer science)" href="https://en.wikipedia.org/wiki/Variable_(computer_science)">variable</a>&nbsp;declarations and&nbsp;<a title="Parameter (computer programming)" href="https://en.wikipedia.org/wiki/Parameter_(computer_programming)">parameter lists</a>&nbsp;have the&nbsp;<a title="Data type" href="https://en.wikipedia.org/wiki/Data_type">data type</a>&nbsp;come after the variable name (and with a&nbsp;<a title="Colon (punctuation)" href="https://en.wikipedia.org/wiki/Colon_(punctuation)">colon</a>&nbsp;separator), similar to&nbsp;<a title="Pascal (programming language)" href="https://en.wikipedia.org/wiki/Pascal_(programming_language)">Pascal</a>&nbsp;and&nbsp;<a title="TypeScript" href="https://en.wikipedia.org/wiki/TypeScript">TypeScript</a>.</p>
<p>Variables in Kotlin can be read-only, declared with the&nbsp;<span class="monospaced">val</span>&nbsp;keyword, or mutable, declared with the&nbsp;<span class="monospaced">var</span>&nbsp;keyword.</p>
<p>Class members are public by default, and classes themselves are final by default, meaning that creating a derived class is disabled unless the base class is declared with the&nbsp;<span class="monospaced">open</span>&nbsp;keyword.</p>
<p>In addition to the&nbsp;<a title="Class (computer programming)" href="https://en.wikipedia.org/wiki/Class_(computer_programming)">classes</a>&nbsp;and&nbsp;<a title="Method (computer programming)" href="https://en.wikipedia.org/wiki/Method_(computer_programming)">methods</a>&nbsp;(called member functions in Kotlin) of object-oriented programming, Kotlin also supports&nbsp;<a title="Procedural programming" href="https://en.wikipedia.org/wiki/Procedural_programming">procedural programming</a>&nbsp;with the use of&nbsp;<a class="mw-redirect" title="Function (computer science)" href="https://en.wikipedia.org/wiki/Function_(computer_science)">functions</a>.&nbsp;Kotlin functions (and constructors) support&nbsp;<a title="Default argument" href="https://en.wikipedia.org/wiki/Default_argument">default arguments</a>,&nbsp;<a title="Variadic function" href="https://en.wikipedia.org/wiki/Variadic_function">variable-length argument</a>&nbsp;lists,&nbsp;<a title="Named parameter" href="https://en.wikipedia.org/wiki/Named_parameter">named arguments</a>&nbsp;and overloading by unique signature. Class member functions are virtual, i.e. dispatched based on the runtime type of the object they are called on.</p>
<p>Kotlin 1.3 adds (experimental) support for contracts&nbsp;(inspired by Eiffel's&nbsp;<a title="Design by contract" href="https://en.wikipedia.org/wiki/Design_by_contract">design by contract</a>&nbsp;programming paradigm)</p>
<h2><span id="Syntax" class="mw-headline">Syntax</span></h2>
<h3><span id="Functional_programming_style" class="mw-headline">Functional programming style</span></h3>
<p>Kotlin relaxes Java's restriction of allowing&nbsp;<a title="Static (keyword)" href="https://en.wikipedia.org/wiki/Static_(keyword)">static</a>&nbsp;methods and variables to exist only within a class body. Static objects and functions can be defined at the top level of the package without needing a redundant class level. For compatibility with Java, Kotlin provides a&nbsp;<code>JvmName</code>&nbsp;annotation which specifies a class name used when the package is viewed from a Java project. For example,&nbsp;<code>@file:JvmName("JavaClassName")</code>.</p>
<h3><span id="Main_entry_point" class="mw-headline">Main entry point</span></h3>
<p>As in C, C++, Java, and Go, the&nbsp;<a title="Entry point" href="https://en.wikipedia.org/wiki/Entry_point">entry point</a>&nbsp;to a Kotlin&nbsp;<a title="Computer program" href="https://en.wikipedia.org/wiki/Computer_program">program</a>&nbsp;is a function named "main", which may be passed an array containing any&nbsp;<a title="Command-line interface" href="https://en.wikipedia.org/wiki/Command-line_interface">command line</a>&nbsp;arguments. (This is optional since Kotlin 1.3).&nbsp;<a title="Perl" href="https://en.wikipedia.org/wiki/Perl">Perl</a>&nbsp;and&nbsp;<a title="Unix shell" href="https://en.wikipedia.org/wiki/Unix_shell">Unix shell</a>&nbsp;style&nbsp;<a title="String interpolation" href="https://en.wikipedia.org/wiki/String_interpolation">string interpolation</a>&nbsp;is supported.&nbsp;<a title="Type inference" href="https://en.wikipedia.org/wiki/Type_inference">Type inference</a>&nbsp;is also supported.</p>

<h3><span id="Extension_methods" class="mw-headline">Extension methods</span></h3>
<p>Similar to C#, Kotlin allows a user to add methods to any class without the formalities of creating a derived class with new methods. Instead, Kotlin adds the concept of an&nbsp;<a title="Extension method" href="https://en.wikipedia.org/wiki/Extension_method">extension method</a>&nbsp;which allows a function to be "glued" onto the public method list of any class without being formally placed inside of the class. In other words, an extension method is a helper method that has access to all the public interface of a class which it can use to create a new method interface to a target class and this method will appear exactly like a method of the class, appearing as part of code completion inspection of class methods. For example:</p>
<div class="mw-highlight mw-highlight-lang-kotlin mw-content-ltr" dir="ltr">
<pre><span class="lineno">1 </span><span class="k">package</span> <span class="nn">MyStringExtensions</span>
<span class="lineno">2 </span>
<span class="lineno">3 </span><span class="k">fun</span> <span class="nc">String</span><span class="p">.</span><span class="nf">lastChar</span><span class="p">():</span> <span class="n">Char</span> <span class="p">=</span> <span class="k">get</span><span class="p">(</span><span class="n">length</span> <span class="p">-</span> <span class="m">1</span><span class="p">)</span>
<span class="lineno">4 </span>
<span class="lineno">5 </span><span class="p">&gt;&gt;&gt;</span> <span class="n">println</span><span class="p">(</span><span class="s">"Kotlin"</span><span class="p">.</span><span class="n">lastChar</span><span class="p">())</span>
</pre>
</div>

<h3><span id="Unpack_arguments_with_spread_operator" class="mw-headline">Unpack arguments with spread operator</span></h3>
<p>Similar to Python, the spread operator asterisk (*) unpacks an array's contents as comma-separated arguments to a function:</p>
<div class="mw-highlight mw-highlight-lang-kotlin mw-content-ltr" dir="ltr">
<pre><span class="lineno">1 </span><span class="k">fun</span> <span class="nf">main</span><span class="p">(</span><span class="n">args</span><span class="p">:</span> <span class="n">Array</span><span class="p">&lt;</span><span class="n">String</span><span class="p">&gt;)</span> <span class="p">{</span>

<h3><span id="Classes_are_public_by_default" class="mw-headline">Classes are public by default</span></h3>
<p>Kotlin provides the following keywords to restrict visibility for top-level declaration, such as classes, and for class members:</p>
<pre>   public, internal, protected, and private.
</pre>
<p>When applied to a class member:</p>
<pre>   public (default): Visible everywhere 
   internal:         Visible in a module 
   protected:        Visible in subclasses 
   private:          Visible in a class
</pre>
<p>When applied to a top-level declaration</p>
<pre>   public (default):  Visible everywhere
   internal:          Visible in a module
   private:           Visible in a file
</pre>
<p>Example:</p>
<div class="mw-highlight mw-highlight-lang-kotlin mw-content-ltr" dir="ltr">

<span class="lineno">5 </span>
<span class="lineno">6 </span>    <span class="c1">// method is visible to current class and derived classes</span>

<h3><span id="Primary_constructor_vs._secondary_constructors" class="mw-headline">Primary constructor vs. secondary constructors</span></h3>
<p>Kotlin supports the specification of a "primary constructor" as part of the class definition itself, consisting of an argument list following the class name. This argument list supports an expanded syntax on Kotlin's standard function argument lists, that enables declaration of class properties in the primary constructor, including visibility, extensibility and mutability attributes. Additionally, when defining a subclass, properties in super-interfaces and super-classes can be overridden in the primary constructor.</p>
<div class="mw-highlight mw-highlight-lang-kotlin mw-content-ltr" dir="ltr">

<h3><span id="Data_Class" class="mw-headline">Data Class</span></h3>

<p>Kotlin makes a distinction between nullable and non-nullable data types. All nullable objects must be declared with a "?" postfix after the type name. Operations on nullable objects need special care from developers: null-check must be performed before using the value. Kotlin provides null-safe operators to help developers:</p>
<ul>
<li><span class="monospaced">?.</span>&nbsp;(<a title="Safe navigation operator" href="https://en.wikipedia.org/wiki/Safe_navigation_operator">safe navigation operator</a>) can be used to safely access a method or property of a possibly null object. If the object is null, the method will not be called and the expression evaluates to null.</li>
<li><span class="monospaced">?:</span>&nbsp;(<a title="Null coalescing operator" href="https://en.wikipedia.org/wiki/Null_coalescing_operator">null coalescing operator</a>) often referred to as the&nbsp;<a title="Elvis operator" href="https://en.wikipedia.org/wiki/Elvis_operator">Elvis operator</a>:</li>
</ul>
<div class="mw-highlight mw-highlight-lang-kotlin mw-content-ltr" dir="ltr">
<pre><span class="lineno">1 </span><span class="k">fun</span> <span class="nf">sayHello</span><span class="p">(</span><span class="n">maybe</span><span class="p">:</span> <span class="n">String</span><span class="p">?,</span> <span class="n">neverNull</span><span class="p">:</span> <span class="n">Int</span><span class="p">)</span> <span class="p">{</span>
<span class="lineno">2 </span>    <span class="c1">// use of elvis operator</span>
<span class="lineno">3 </span>    <span class="k">val</span> <span class="py">name</span><span class="p">:</span> <span class="n">String</span> <span class="p">=</span> <span class="n">maybe</span> <span class="o">?:</span> <span class="s">"stranger"</span>
<span class="lineno">4 </span>    <span class="n">println</span><span class="p">(</span><span class="s">"Hello $name"</span><span class="p">)</span>
<span class="lineno">5 </span><span class="p">}</span>
</pre>
</div>
<p>An example of the use of the safe navigation operator:</p>
<div class="mw-highlight mw-highlight-lang-kotlin mw-content-ltr" dir="ltr">
<pre><span class="lineno">1 </span><span class="c1">// returns null if...</span>
<span class="lineno">2 </span><span class="c1">// - foo() returns null,</span>
<span class="lineno">3 </span><span class="c1">// - or if foo() is non-null, but bar() returns null,</span>
<span class="lineno">4 </span><span class="c1">// - or if foo() and bar() are non-null, but baz() returns null.</span>
<span class="lineno">5 </span><span class="c1">// vice versa, return value is non-null if and only if foo(), bar() and baz() are non-null</span>
<span class="lineno">6 </span><span class="n">foo</span><span class="p">()</span><span class="o">?.</span><span class="n">bar</span><span class="p">()</span><span class="o">?.</span><span class="n">baz</span><span class="p">()</span>
</pre>
</div>
<p>Kotlin provides support for&nbsp;<a class="mw-redirect" title="Higher order functions" href="https://en.wikipedia.org/wiki/Higher_order_functions">higher order functions</a>&nbsp;and&nbsp;<a class="mw-redirect" title="Anonymous functions" href="https://en.wikipedia.org/wiki/Anonymous_functions">anonymous functions</a>&nbsp;or lambdas.</p>
<div class="mw-highlight mw-highlight-lang-kotlin mw-content-ltr" dir="ltr">
<pre><span class="lineno">1 </span><span class="c1">// the following function takes a lambda, f, and executes f passing it the string, "lambda"</span>
<span class="lineno">2 </span><span class="c1">// note that (s: String) -&gt; Unit indicates a lambda with a String parameter and Unit return type</span>
<span class="lineno">3 </span><span class="k">fun</span> <span class="nf">executeLambda</span><span class="p">(</span><span class="n">f</span><span class="p">:</span> <span class="p">(</span><span class="n">s</span><span class="p">:</span> <span class="n">String</span><span class="p">)</span> <span class="p">-&gt;</span> <span class="n">Unit</span><span class="p">)</span> <span class="p">{</span>
<span class="lineno">4 </span>    <span class="n">f</span><span class="p">(</span><span class="s">"lambda"</span><span class="p">)</span>
<span class="lineno">5 </span><span class="p">}</span>
</pre>
</div>
<p>Lambdas are declared using braces,&nbsp;<span class="monospaced">{ }&nbsp;</span>. If a lambda takes parameters, they are declared within the braces and followed by the&nbsp;<span class="monospaced">-&gt;</span>&nbsp;operator.</p>
<div class="mw-highlight mw-highlight-lang-kotlin mw-content-ltr" dir="ltr">
<pre><span class="lineno">1 </span><span class="c1">// the following statement defines a lambda that takes a single parameter and passes it to the println function</span>
<span class="lineno">2 </span><span class="k">val</span> <span class="py">l</span> <span class="p">=</span> <span class="p">{</span> <span class="n">c</span> <span class="p">:</span> <span class="n">Any</span><span class="p">?</span> <span class="p">-&gt;</span> <span class="n">println</span><span class="p">(</span><span class="n">c</span><span class="p">)</span> <span class="p">}</span>
<span class="lineno">3 </span><span class="c1">// lambdas with no parameters may simply be defined using { }</span>
<span class="lineno">4 </span><span class="k">val</span> <span class="py">l2</span> <span class="p">=</span> <span class="p">{</span> <span class="n">print</span><span class="p">(</span><span class="s">"no parameters"</span><span class="p">)</span> <span class="p">}</span>
</pre>
</div>

<h2><span id="Applications" class="mw-headline">Applications</span></h2>
<p>Kotlin is widely used for&nbsp;<a title="Android (operating system)" href="https://en.wikipedia.org/wiki/Android_(operating_system)">Android</a>&nbsp;development. The platform was stuck on Java 7 for a while (with some contemporary language features made accessible through the use of Retrolambda&nbsp;or the Jack toolchain) and Kotlin introduces many improvements for programmers such as null-pointer safety, extension functions and infix notation. Accompanied by full Java compatibility and good IDE support (<em>Android Studio</em>) it is intended to improve code readability, give an easier way to extend Android SDK classes and speed up development.</p>
<p>Kotlin was announced as an official Android development language at&nbsp;<a title="Google I/O" href="https://en.wikipedia.org/wiki/Google_I/O">Google I/O</a>&nbsp;2017. It became the third language fully supported for Android, in addition to Java and C++.</p>
